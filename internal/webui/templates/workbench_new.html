Nuova migrazione

Definisci la migrazione di un account: da quale server parti, verso quale server vai, quale account copi e cosa vuoi spostare. Alla fine ti guideremo al controllo preliminare (preflight).

1 · Nome della migrazione
2 · Da quale server stai migrando? (sorgente)

Il server di partenza viene solo letto: non viene mai modificato.

3 · Verso quale server vuoi migrare? (destinazione)

Il server di destinazione è quello che riceve la copia.

4 · Cosa vuoi migrare?

Scegli i contenuti da copiare. Nessuna opzione «migra tutto»: selezioni solo ciò che ti serve.

DNS — area delicata

Il DNS è separato di proposito: modificarlo può raggiungere i nameserver di produzione e cambiare dove punta il dominio per tutti. Non è mai incluso automaticamente in una migrazione di contenuti. Attivalo solo se sai di volerlo gestire da qui.

5 · Pronto per il preflight

Alla creazione ti porteremo alla migrazione, con il controllo preliminare (preflight) come prossimo passo consigliato.

Configurazione tecnica richiesta prima del preflight. Le credenziali (password SSH) non si inseriscono qui: vanno completate nel file host.yaml nella directory di lavoro. Puoi configurarle dalla pagina delle connessioni.
